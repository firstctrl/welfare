# Loan Statement Feature — Design Spec

**Date:** 2026-05-23  
**Status:** Approved

## Overview

Add a Loan Statement panel to the Reports page. User selects a staff member (filtered to borrowers only), selects one of their loans (with status tag), then views a statement with KPI summary cards and a full instalment-by-instalment repayment history. Statement can be downloaded as PDF or sent to the staff member's email.

Mirrors the existing Contribution Statement feature in structure and UX pattern.

---

## Architecture

### Backend — `reports` module

Three new endpoints added to `reports.controller.ts` and implemented in `reports.service.ts`.

One supporting endpoint added to fetch borrowers list.

No changes to `loans.controller.ts` or `loans.service.ts`.

### Frontend — Reports page

New `LoanStatementPanel` component added to `reports-client.tsx`. New sidebar nav item. New client functions in `reports.ts`.

## API Design

### 1. Borrowers List

```
GET /reports/loans/borrowers
```

Returns staff who have at least one loan (any status), sorted by display name.

**Response:**
```ts
Array<{
  staffId: string;
  staffNo: string;
  displayName: string;
}>
```

Implementation: distinct `staffId` values from `Loan` collection, populate names from staff collection.

---

### 2. Loan Statement — JSON

```
GET /reports/loans/staff-statement?staffId={id}&loanId={id}
```

Validates that the loan belongs to the given staff member. Returns full statement data.

**Response:**
```ts
{
  staff: {
    staffNo: string;
    displayName: string;
    department: string;
  };
  loan: {
    id: string;
    principalAmount: number;
    interestRate: number;
    totalRepayable: number;
    tenureMonths: number;
    disbursedDate: string; // ISO date
    status: 'Active' | 'Completed' | 'WrittenOff' | 'BadDebt' | 'Defaulted';
    chequeNo?: string;
    pvNo?: string;
    guarantor: {
      staffNo: string;
      displayName: string;
    };
  };
  kpis: {
    totalPaid: number;
    outstanding: number;
    penaltyPaid: number;
    completionRate: number; // percentage, instalments Paid/Waived ÷ total
  };
  instalments: Array<{
    instalmentNumber: number;
    dueDate: string;
    dueAmount: number;
    principalAmount: number;
    interestAmount: number;
    paidAmount: number;
    penaltyAmount: number;
    paidDate?: string;
    status: 'Pending' | 'Partial' | 'Paid' | 'Overdue' | 'Waived';
    source?: 'DirectPayment' | 'ExitDeduction' | 'GuarantorOffset';
  }>;
}
```

---

### 3. Loan Statement — PDF Download

```
GET /reports/loans/staff-statement/pdf?staffId={id}&loanId={id}
```

Calls `getLoanStatement()` internally, renders `loan-statement.template.tsx` via Puppeteer, streams PDF response.

**Response:** `application/pdf` with `Content-Disposition: attachment; filename="loan-statement-{staffNo}-{loanId}.pdf"`

---

### 4. Loan Statement — Send Email

```
POST /reports/loans/staff-statement/send
Body: { staffId: string; loanId: string }
```

Generates PDF, renders `loan-statement-email.template.tsx`, sends via existing email service to staff's registered email address.

**Response:** `{ message: 'Statement sent successfully' }`

---

## Service Methods (`reports.service.ts`)

### `getLoanStatement(staffId: string, loanId: string)`

1. Fetch loan by `loanId`, assert `loan.staffId === staffId` (throw 400 if mismatch).
2. Populate staff details and guarantor details from staff collection.
3. Fetch all `LoanRepayment` records for `loanId`, ordered by `instalmentNumber`.
4. Compute KPIs:
   - `totalPaid` = sum of `paidAmount` across all repayments
   - `outstanding` = `loan.totalRepayable - totalPaid` (floor at 0)
   - `penaltyPaid` = sum of `penaltyAmount`
   - `completionRate` = (count of Paid + Waived instalments) / `tenureMonths` × 100
5. Return shaped response object.

### `generateLoanStatementPdf(staffId, loanId)`

Calls `getLoanStatement()`, renders HTML template, runs Puppeteer, returns Buffer.

## Templates

### `loan-statement.template.tsx` (PDF)

Puppeteer-rendered HTML. Structure mirrors `contribution-statement.template.tsx`:

- **Header band** (blue): org name, logo, "LOAN STATEMENT" title, "CONFIDENTIAL" watermark
- **Staff + Loan info table**: Staff No, Name, Department | Principal, Rate, Tenure, Disbursed Date, Status, Cheque No, PV No, Guarantor
- **KPI cards row**: Principal · Total Repayable · Amount Paid · Outstanding · Penalty Paid
- **Instalment table**:

  | # | Due Date | Due Amount | Principal | Interest | Paid | Penalty | Paid Date | Status |

  Status cells color-coded:
  - Paid: green (`#16a34a`)
  - Partial: orange (`#d97706`)
  - Overdue / Pending: red (`#dc2626`)
  - Waived: gray (`#6b7280`)

- **Footer**: Generation timestamp, "Generated by Welfare System"

### `loan-statement-email.template.tsx` (Email)

React Email component. Structure:
- Org header
- Staff + loan summary table
- KPI summary box
- Instalment table (same columns as PDF)
- Footer with timestamp

## Frontend UI

### Sidebar Nav

New item added to Reports sidebar: **Loan Statement** (between Contribution Statement and Bulk Statements, or grouped with loan reports — implementation decision).

### `LoanStatementPanel` Component

Added to `reports-client.tsx`.

**Step 1 — Staff selector**
- Autocomplete input
- Fetches `GET /reports/loans/borrowers` on mount
- Placeholder: "Search borrower..."

**Step 2 — Loan selector**
- Appears after staff selected
- Fetches `GET /loans?staffId=&limit=100`
- Each option: `{amount} · {disbursedDate} · [{STATUS}]`
- Status badge colors: Active=blue, Completed=green, WrittenOff=gray, BadDebt=red, Defaulted=orange

**Step 3 — Statement view**
Appears after loan selected. Fetches `GET /reports/loans/staff-statement?staffId=&loanId=`.

*Loan header block:*
Disbursed date · Tenure (N months) · Interest rate (N%) · Guarantor name · Cheque No · PV No · Status badge

*KPI cards (5):*
| Principal | Total Repayable | Amount Paid | Outstanding | Penalty Paid |

*Instalment table:*
`# | Due Date | Due Amount | Principal | Interest | Paid | Penalty | Paid Date | Status`

Color-coded status cells matching PDF template.

*Action buttons:*
- **Download PDF** → triggers `downloadLoanStatementPdf(staffId, loanId)`
- **Send Statement** (visible only when permission = `full`) → calls `sendLoanStatement(staffId, loanId)` with toast feedback

## Client Library (`reports.ts`)

Four new functions:

```ts
getLoanBorrowers(): Promise<BorrowerOption[]>
getLoanStatement(staffId: string, loanId: string): Promise<LoanStatementResponse>
downloadLoanStatementPdf(staffId: string, loanId: string): Promise<void>
sendLoanStatement(staffId: string, loanId: string): Promise<void>
```

`downloadLoanStatementPdf` fetches the PDF endpoint and triggers browser download (same pattern as `downloadStatementPdf` for contributions).

---

## Error Handling

| Scenario | Behavior |
|---|---|
| `loanId` does not belong to `staffId` | API returns 400, UI shows error toast |
| Staff has no email on record | `send` endpoint returns 400, UI shows error toast |
| PDF generation fails | API returns 500, UI shows error toast |
| No loans found for staff | Borrowers list excludes them (filtered at source) |
